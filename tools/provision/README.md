# EspScreen Provisioner

One-shot CLI tool — push Claude OAuth credentials (and optional WiFi) to an EspScreen ESP32 over USB serial.

## Requirements

- Node.js ≥ 18
- `serialport` npm package (auto-installed on first `npm install`)
- EspScreen firmware running on the ESP32
- `~/.claude/.credentials.json` present and current (generated by `claude login`)

## Install

```sh
cd C:\git\EspScreen\tools\provision
npm install
```

## Usage

### First-time setup

```sh
node provision.js
```

Auto-detects the ESP32 COM port, reads credentials, sends provisioning commands, starts polling.

### With WiFi provisioning

```sh
node provision.js --wifi-ssid "YourNetwork" --wifi-pass "yourpassword"
```

### Specify port manually

```sh
node provision.js --port COM20
```

### Custom profile label

```sh
node provision.js --label "Work Account"
```

### List available serial ports

```sh
node provision.js --list-ports
# or: npm run list
```

### Reset a crashed/wedged board before provisioning

```sh
node provision.js --reset
```

Toggles DTR/RTS to power-cycle the board via its EN pin before sending any commands. Equivalent to pressing the reset button. Safe to combine with other flags:

```sh
node provision.js --reset --port COM20 --label "Work Account"
```

### Refresh tokens (after re-login on host)

```sh
node provision.js
```

Idempotent — detects existing profile and updates tokens without creating a duplicate.

## Options

| Flag | Default | Description |
|------|---------|-------------|
| `--port <COMx>` | auto-detect | Override serial port |
| `--label <name>` | `"Default"` | Claude profile label on board |
| `--credentials <path>` | `~/.claude/.credentials.json` | Override creds file path |
| `--wifi-ssid <ssid>` | — | WiFi SSID (requires --wifi-pass) |
| `--wifi-pass <pass>` | — | WiFi password |
| `--list-ports` | — | Enumerate serial ports and exit |
| `--reset` | off | Power-cycle board via DTR/RTS before provisioning |
| `-h`, `--help` | — | Show help |

## Port Auto-Detection

Scans all serial ports for known ESP32 USB bridge chips:

| Chip | VID | Notes |
|------|-----|-------|
| WCH CH340 | `1A86` | Most common clone dev kits |
| Silicon Labs CP210x | `10C4` | Adafruit, SparkFun |
| FTDI FT232 | `0403` | Older boards |
| Espressif built-in | `303A` | ESP32-S3, C3 native USB |

If multiple matches, picks the highest COM number (DevKits often appear at COM20+).

## Token Security

Access and refresh tokens are never printed in full. The script logs a redacted form:
```
eyJ...abc123
```

## Error Handling

| Error | Action |
|-------|--------|
| No credentials file | Exits with message to run `claude login` |
| No ESP32 port found | Lists known VIDs, suggests `--list-ports` |
| Port in use | Suggests closing VS Code Serial Monitor / other tools |
| 401 from poll | Tells you to re-run `claude login` on host, then re-provision |
| Token expired | Flagged in summary with exact expiry info |

## Board Commands Sent

```
claude profile list
claude profile add "<label>"      # skipped if label exists
claude token set "<label>" <accessToken> <refreshToken> <expiresAtSec>
claude profile use "<label>"
wifi add "<ssid>" "<pass>"        # only if --wifi-ssid/--wifi-pass provided
claude poll                       # waits up to 12s for HTTPS handshake
```
